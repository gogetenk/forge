# Contributing to Forge

Thanks for your interest in contributing to Forge.

## How to contribute

1. **Open an issue first** for significant changes to discuss the approach
2. Fork the repository
3. Create a feature branch (`feat/your-feature`)
4. Make your changes
5. Submit a PR towards `master`

## What we're looking for

- **Stack examples** (`examples/`) — Show Forge working with your stack
- **Hook improvements** — Better detection patterns, fewer false positives
- **Documentation** — Guides, tutorials, translations
- **Bug reports** — Especially edge cases from real-world usage

## Guidelines

- Keep documentation clear and concise
- Hooks must work on Linux, macOS, and Windows (Git Bash)
- Agent definitions should be stack-agnostic where possible
- Test your hooks before submitting
