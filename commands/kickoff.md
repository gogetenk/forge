# /kickoff — Bootstrap a new project with the PO agent

Read `agents/po.md` and execute the kickoff process:

1. Ask the human to describe their project (what, who, why)
2. Lead a structured Q&A to understand all modules, roles, business rules
3. Propose a module breakdown — get validation
4. Write all .feature files (pure natural language, zero tech)
5. Create all task files (todo-*.md) with dependencies
6. Present the complete backlog with dependency graph
7. Get human validation — then ready for `/forge`

DO NOT skip the Q&A. DO NOT assume business rules. ASK until you have enough to write specs.
